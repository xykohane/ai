1、ECharts 图表库
ECharts 是百度开源的数据可视化库，支持柱状图、折线图、饼图、散点图、地图等几十种图表类型。在 Vue3 中使
用 ECharts 的基本流程：
安装 → 获取 DOM 容器 → 初始化图表实例 → 设置配置项 → 监听窗口变化 → 销毁时清理
1.1 安装与引入
npm install echarts
<script setup>
import * as echarts from 'echarts'
</script>
为什么用 import * as echarts？ ECharts 采用"按需引入"的打包方式，import * 会引入所有图表类型。
生产环境可以用 echarts/core 按需引入减小包体积，开发阶段全量引入更方便。

1.3 饼图渲染逻辑
<script setup lang="ts">
import { ref, onMounted, onBeforeUnmount, nextTick } from 'vue'
import * as echarts from 'echarts'
import { getDashboardData } from '../../api/dashboard'
import type { DashboardData } from '../../types'
const data = ref<DashboardData | null>(null)
const pieChartRef = ref<HTMLElement>() // DOM 容器引用
let pieChart: echarts.ECharts | null = null // ECharts 实例
// 获取仪表盘数据
async function fetchData() {
const res = await getDashboardData()
data.value = res.data
// 等待 DOM 更新后再渲染图表
await nextTick()
renderPieChart()
}
// 渲染饼图
function renderPieChart() {
// 防御性检查：DOM 容器和数据都存在才渲染
if (!pieChartRef.value || !data.value?.skillDistribution?.length) return
// 1. 在 DOM 元素上初始化 ECharts 实例
pieChart = echarts.init(pieChartRef.value)
// 2. 设置图表配置项
pieChart.setOption({
tooltip: { trigger: 'item' },
series: [{

type: 'pie',
radius: '60%',
data: data.value.skillDistribution,
emphasis: {
itemStyle: {
shadowBlur: 10,
shadowOffsetX: 0,
shadowColor: 'rgba(0, 0, 0, 0.5)'
}
}
}]
})
}
// 窗口大小变化时重绘图表
function handleResize() {
pieChart?.resize()
}
onMounted(() => {
fetchData()
window.addEventListener('resize', handleResize)
})
onBeforeUnmount(() => {
// 销毁图表实例，释放 Canvas 和内存
pieChart?.dispose()
// 移除事件监听，防止内存泄漏
window.removeEventListener('resize', handleResize)
})
</script>
ECharts 使用五步法：
步骤 代码 说明
1. 获取 DOM ref<HTMLElement>() 模板中用 ref="pieChartRef" 绑定
2. 初始化 echarts.init(dom) 在 DOM 元素上创建图表实例
3. 配置选项 chart.setOption({...}) 设置图表类型、数据、样式
4. 响应式 addEventListener('resize', ...) 窗口大小变化时重绘
5. 清理资源 chart.dispose() 组件销毁时释放内存
nextTick 的重要性： data.value = res.data 更新数据后，Vue 不会立即更新 DOM，而是异步批量更
新。await nextTick() 等待 DOM 更新完成，此时 pieChartRef.value 才能正确获取到 DOM 元素。如果
不等 nextTick，pieChartRef.value 可能是 null，echarts.init(null) 会报错。

2.3 Object.assign 填充表单
Day08 回顾： reactive() 创建的对象不能直接重新赋值（form = { ... } 会断开 Proxy 包装），必须用
Object.assign 逐个修改属性。这个坑在 Day08 的 Composition API 部分已经讲过，这里再次出现是因为技
能树编辑功能需要"回填数据到表单"。
// 编辑模式：将技能数据复制到表单中
Object.assign(form, {
name: skill.name,
category: skill.category,
description: skill.description,
parentId: skill.parentId,
icon: skill.icon
})
为什么不直接 form = { ... } ？ reactive() 创建的对象不能直接重新赋值，否则会丢失响应性（Proxy
包装会断开）。必须用 Object.assign 逐个修改属性，或者改用 ref 包装后修改 .value。

4、资源清理
Dashboard 组件销毁时必须清理 ECharts 实例和事件监听：
onBeforeUnmount(() => {
pieChart?.dispose() // 销毁图表实例
window.removeEventListener('resize', handleResize) // 移除事件监听
})
清理操作 原因
chart.dispose() 释放 ECharts 占用的内存和 Canvas 资源
removeEventListener 防止已销毁组件的回调函数被触发（内存泄漏）
和 HarmonyOS 的对比： Vue 的 onBeforeUnmount ≈ HarmonyOS 的 aboutToDisappear。两者都需要在
组件销毁时清理定时器、事件监听等资源，否则会造成内存泄漏。这是所有 UI 框架的通用最佳实践。
5、Vue3 开发模式总结
Vue3 后台管理系统的完整开发模式：
开发流程：
1. views/ 下创建页面组件（.vue 文件）
2. api/ 下封装后端接口调用
3. stores/ 下管理全局状态
4. router/index.js 注册路由
页面结构：
<template> → HTML 模板（Element Plus 组件）
<script setup> → 业务逻辑（Composition API）
<style scoped> → 样式（scoped 限定作用域，不污染其他组件）
数据流：
用户操作 → 调用 API → Axios 拦截器注入 Token
→ 后端处理 → 响应拦截器解包
→ 更新 State（Pinia / ref）
→ UI 自动更新（Vue 响应式系统）
scoped 样式： <style scoped> 中的样式只对当前组件生效，不会影响其他组件。Vue 在编译时会给当前组
件的 DOM 元素添加唯一的 data-v-xxx 属性，CSS 选择器也会加上对应的属性选择器，实现样式隔离。
